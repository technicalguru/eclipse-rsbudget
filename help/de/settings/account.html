Konto-Einstellungen

Dieser Reiter listet die Bankkonten für ihre RsBudget-Installation auf. Sie können beliebige Konten anlegen, verändern oder auch wieder lö,schen.

Bankkonten haben derzeit keine tiefere Bedeutung für das Programm. Sie können also ein fiktives Konto eintragen. Jede buchung muss jedoch mit einem Konto verknüpft sein, um zukünftige Erweiterungen zu ermöglichen, z.B. um Buchungen per HBCI von Ihrer Bank automatisch abzufragen.

Konto anlegen

Klicken Sie auf "Neues Konto anlegen". Dadurch wird ein neuer Eintrag erzeugt und zur Bearbeitung selektiert. Die weitere Bearbeitung ist identisch zum Konto ändern.

Konto ändern

Die folgenden Eigenschaften eines Kontos können festgelegt werden:

Eigenschaft Beschreibung
Name Jedes Konto wird durch einen Namen identifiziert. Benutzen Sie einen sprechenden Namen für Ihr Konto.
Kontoinhaber Der Inhaber des jeweiligen Kontos. In den meisten Fällen wird das Ihr eigener Name sein.
Kontonummer Die Kontonummer bei der Bank.
BLZ Die Bankleitzahl. Wählen Sie einfach Ihre Bank aus. Falls Ihre Bank nicht aufgeführt ist (da nur deutsche Banken gelistet sind), wählen Sie einfach eine beliebige Bank aus.
Default Dieses Flag markiert das Konto als Standard-Konto. RsBudget assoziiert alle Buchungen automatisch mit diesem Konto. Für individuelle Buchungen kann das assoziierte Konto dann verändert werden.
Planungsrelevant Dieses Konto wird bei der Planung berücksichtigt. Wenn Sie dieses Flag löschen, wird das Konto nicht mehr für statistische oder andere Zwecke berücksichtigt. (z.B. bei Aktiendepots).

Konto löschen

Klicken Sie auf "Konto löschen". RsBudget wird Sie nach einer Bestätigung fragen und das Konto dann aus der Liste löschen. Das Konto wird jedoch nicht physisch aus der Datenbank gelöscht, sondern lediglich als inaktiv gekennzeichnet. Sie können die Löschung rückgängig machen, wenn Sie den Dialog mit "Cancel" abbrechen. Nur wenn Sie den Einstellungs-Dialog mit "OK" bestätigen, wird die Deaktivierung des Kontos auch vollzogen. Derzeit gibt es keine Möglichkeit, ein deaktiviertes Konto wieder zu reaktivieren (Teil einer zukünftigen Erweiterung).
